# 🧪 Классификация таблеток с помощью нейронной сети

Этот проект посвящён построению и обучению нейросетевой модели для классификации таблеток по изображению.
В ноутбуке последовательно выполняются этапы подготовки данных, построения модели, обучения, оценки качества и анализа ошибок.

```python
├── notebook__у.ipynb      # основной ноутбук проекта
├── data/                  # изображения и метки
├── README.md              # текущий файл
```


## Описание задачи

Задача проекта — автоматизировать процесс распознавания таблеток по изображению.
Модель должна уметь принимать входное изображение и относить его к одному из заранее заданных классов.

В рамках исследования проведены:

- настройка вычислительной среды,
- подготовка набора данных,
- построение и обучение нейронной сети,
- оценка полученных результатов с помощью метрик Accuracy, Precision, Recall, F1,
- визуализация процесса обучения,
- анализ ошибочных предсказаний.

## Используемые библиотеки

В ноутбуке подключаются ключевые Python-библиотеки:

- `torch`, `torchvision` — создание и обучение нейронной сети
- `numpy`, `pandas` — обработка данных
- `matplotlib`, `seaborn` — графики и визуализация
- `sklearn` — разбиение данных и метрики
- `tqdm` — прогресс-бары
- и другие вспомогательные инструменты

## Подготовка данных

Выполняются следующие шаги:

- настройка путей и параметров окружения,
- загрузка изображений и меток,
- применение преобразований (`transforms`) для повышения качества обобщения,
- формирование обучающей и тестовой выборок,
- создание `DataLoader`-ов для удобной итерации.

## Построение модели

В ноутбуке представлена архитектура нейросети (на основе `torch.nn`), включающая:

- сверточные блоки (`Conv` + `ReLU` + `Pool`),
- полносвязные слои,
- слои нормализации,
- дропаут,
- классификатор на выходе.

Модель оптимизируется с использованием:

- `Adam` или `SGD`,
- функции потерь `CrossEntropyLoss`.

## Обучение

В процессе обучения выводятся:

- значения функции потерь на train/test,
- ключевые метрики,
- графики:
  - loss,
  - accuracy.

Также предусмотрена автоматическая проверка устройства:
если доступен CUDA, обучение переносится на GPU.

## Оценка качества

После обучения вычисляются метрики:

- Accuracy
- Precision
- Recall
- F1-score

Формируется таблица результатов и строятся наглядные графики.

## Анализ ошибок

В ноутбуке проводится анализ случаев, когда модель ошиблась:

- отображение изображений с неверными предсказаниями,
- сравнение истинных и предсказанных меток,
- обнаружение закономерностей для дальнейшего улучшения модели.
