# Пайплайн обучения LLM: Pretrain с нуля + SFT


Два ключевых этапа жизненного цикла языковой модели на практике — от обучения собственной модели с нуля до превращения готовой LLM в отвечающего ассистента - в условиях одной потребительской видеокарты.

## Цели

Пройти оба этапа обучения LLM:

1. **Pretrain** - обучить с нуля собственную модель архитектуры LLaMA (~132M параметров) на русской классической литературе.
2. **Post-train SFT** - дообучить готовую `Qwen2.5-0.5B` на инструктивном датасете, чтобы модель отвечала на русскоязычные вопросы.

Полноценный претрейн даже небольшой модели требует тысяч GPU-часов, поэтому этап 1 выполнен в упрощённом виде (демонстрация механики), а эффект SFT показан на сильной базовой модели, где он виден за разумное время.

## Данные

| Этап | Датасет | Объём |
|---|---|---|
| Pretrain | [RussianNovels](https://github.com/JoannaBy/RussianNovels) — корпус русской прозы | 108 текстов (классика), после дедупликации |
| SFT | [`d0rj/alpaca-cleaned-ru`](https://huggingface.co/datasets/d0rj/alpaca-cleaned-ru) — пары «инструкция → ответ» | ~50k примеров (использовано 10k train / 500 eval) |

Препроцессинг претрейна: перебор кодировок, Unicode-нормализация, фильтрация строк по доле кириллицы, разбивка на предложения и дедупликация, нарезка на чанки под контекст 512 токенов.

## Подход

**Этап 1 - Pretrain (LLaMA с нуля)**
- Собственный **BPE-токенизатор**, словарь 3 000 (обоснованно мал: узкий домен + ограничения железа).
- Архитектура `LlamaConfig`: `hidden_size=1024`, 16 слоёв, 16 голов внимания, 8 KV-голов (**GQA**), RoPE, SiLU, контекст 512.
- Обучение: 3 эпохи, эффективный батч 64 (`batch=8 × grad_accum=8`), fp16, `lr=3e-4` с cosine-шедулером и warmup, режим CLM (предсказание следующего токена).

**Этап 2- SFT (Qwen2.5-0.5B)**
- Дообучение через `trl.SFTTrainer` на форматированных диалогах (system / user / assistant).
- 1 эпоха, эффективный батч 32 (`batch=2 × grad_accum=16`), `lr=2e-5`, cosine. Обучение в **float32** (в fp16 лосс уходил в NaN).
- Замер генерации до и после SFT на одинаковых вопросах — наглядное сравнение.

## Результаты

| Этап | Модель | Итог |
|---|---|---|
| Pretrain | LLaMA ~132M (своя) | train loss ≈ 4.08 за 3 эпохи (~23 мин); модель освоила морфологию, согласование, пунктуацию и литературный регистр |
| SFT | Qwen2.5-0.5B (494M) | train loss ≈ 1.39 за 1 эпоху (~51 мин); после дообучения отвечает по-русски и следует структуре ответа |

Качественное сравнение SFT (до - после): на вопрос о числе планет базовая модель отвечала с ошибками согласования и неверным фактом, после SFT - корректные «8 планет» и связный ответ. Цель этапа - *«факты могут быть ошибочными, но язык русский и структура ответа сохраняется»* - достигнута.

## Выводы и что дальше

- Претрейн-модель учит **язык**, но не следует инструкциям - ожидаемое поведение для этого этапа.
- Узкие места - экономия ресурсов: маленький словарь (3000) дробит редкие слова, мало эпох и данных.
- Улучшения: увеличить словарь, добавить 2–3 эпохи и данных в претрейн; для SFT - обучать на полном датасете, понизить температуру генерации (~0.3), чаще валидировать.

## Запуск

Часть библиотек не поддерживается на Windows нативно - проект выполнялся в WSL2.


### Небольшой гайд для постановки wsl:

1. PowerShell от администратора

```bash
wsl --install
```

2. Вводим имя пользователя и пароль. Пароль не отображается, так что не стоит по несколько раз нажимать на кнопки, они учитываются. А то я подумал, что клавиатура сломалась и процесс завис

3. WSL иногда не резолвит адреса. Сразу фиксим:

```bash
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf
```

4. Изначально ставится новая версия Ubuntu, у меня была 26.04 - она шла с версией Python 3.14 и PyTorch его не воспринимал.. поэтому ставим более позднюю версию питона 3.11 - с такой работаю на windows ей доверяю.

```bash
sudo add-apt-repository ppa:deadsnakes/ppa -y
sudo apt update
sudo apt install python3.11 python3.11-venv python3.11-dev -y
```

5. Создаем виртуальное окружение

```bash
cd ~
python3.11 -m venv dl_env311
source dl_env311/bin/activate
pip install --upgrade pip
```

6. И ставим библиотеки - с этим было много возни, из-за совместимостей, поэтому вот на этих я и остановился.

```bash
pip install torch==2.5.1+cu121 torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121
pip install transformers==5.9.0 datasets accelerate==1.13.0 peft==0.19.1 trl==1.4.0 bitsandbytes==0.49.2 jupyter ipykernel tokenizers unsloth
```
*p.s. возможно что-то дополнительно ставил уже в ноутбуке (уже не помню), ну там подскажет терминал, чего не хватит*

7. Я работаю в VScode, поэтому в ней ставлю расширение **WSL**
8. Жмём `Ctrl+shift+p` > `WSL: Connect to WSL using Distro` > `Ubuntu`.
Открываем папку `File - Open Folder` > В строке вводим папку, либо просто новую создаем, я использовал локалку так удобнее для меня, вот мой путь: `/mnt/d/code/ya_neiro/7. Большие языковые модели (LLM).`

9. Открываем терминал `Ctrl + ~` и активируем окружение

```bash
source ~/dl_env311/bin/activate
python -m ipykernel install --user --name=dl_env311 --display-name "Python DL (WSL)" # это мы ядро зарегали в скобках название мое
```

10. Запускаем Jupiter

```bash
jupyter notebook --no-browser --ip=0.0.0.0 --port=8888
```

11. Там (в терминале) выведется информация по ноутбуку, нужно будет копирнуть токен. Выводится два вида, на нужен именно такой - `http://127.0.0.1:8888/?token=...`
12. В VScode **Выбор ядра** > **Существующий сервер Jupyter** > вставляем ссылку
12. Выбери ядро - **Python DL (WSL)** 
13. Запускаем и работаем

*P.S. WSL после перезагрузки у меня теряет DNS. И на повторный запуск используем*
```bash
echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf
