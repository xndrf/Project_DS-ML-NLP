# Оценка калорийности блюд по фото и списку ингредиентов

Мультимодальная нейросеть для оценки калорийности блюд на основе двух типов данных:
- **Фотография блюда** (RGB-изображение)
- **Список ингредиентов** (текстовые описания)

Модель может быть использована в фитнес- и health-приложениях для автоматического подсчёта калорий.

## Цель проекта

Разработать DL-модель, способную оценить калорийность блюда по его фотографии и списку ингредиентов.

## Целевая метрика

**MAE (Mean Absolute Error)** — средняя абсолютная ошибка в килокалориях.

## Структура датасета

### 1. `data/ingredients.csv`

| Колонка | Описание |
|---------|----------|
| `id` | ID ингредиента |
| `ingr` | Название ингредиента |



### 2. `data/dish.csv`

| Колонка | Описание |
|---------|----------|
| `dish_id` | ID блюда |

### 3. `data/images/`
Директория с фотографиями блюд. Каждая поддиректория соответствует `dish_id` и содержит файл `rgb.png`.

## Архитектура решения

### Анализ данных

- **Сильная правосторонняя асимметрия** калорийности (медиана ~200-300 ккал, выбросы до 1000+ ккал)
- Применено **логарифмирование целевой переменной** для стабилизации обучения
- Использован **оверсемплинг редких классов** (блюд с высокой калорийностью)
- Внедрён расчёт **плотности калорий** (ккал/100г) для учёта вариативности массы порций (от 5 до 3000 г)

### Выбранные модели

| Модальность | Модель | Описание |
|-------------|--------|----------|
| Текст | `microsoft/deberta-v3-small` | Лёгкая, но эффективная модель для понимания языка |
| Изображение | `convnext_tiny` | Современная архитектура для компьютерного зрения |

### Особенности обучения

- **Мультимодальный фьюжн** с кросс-вниманием между текстовыми и визуальными эмбеддингами
- **Оптимизатор**: AdamW с дифференцированными скоростями обучения
- **Шедулер**: CosineAnnealingWarmRestarts
- **Регуляризация**: Dropout, LayerNorm, gradient clipping
- **Ранняя остановка**: при отсутствии улучшений в течение 5 эпох
- **Разморозка** только верхних слоёв предобученных энкодеров

### Выводы и рекомендации

- достигнут целевой показатель MAE < 50 ккал
- применены эффективные техники регуляризации и стабилизации обучения

Для улучшения результатов необходимо:
- увеличение датасета
- добавить больше примеров блюд с калорийностью > 500 ккал
- увеличить общее количество фотографий
- выполнить дополнительно стратификацию данных
- учесть проблему "пустых тарелок" и очень больших порций
- расщирить данные путем добавления время приёма пищи, ввода категорий блюд (суп, салат, горячее и т.д.)
- добавить рассчёт БЖУ (белки, жиры, углеводы) на основе ингредиентов и массы
