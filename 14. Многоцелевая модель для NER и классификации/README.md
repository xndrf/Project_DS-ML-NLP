# NEREL MultiTask — Многозадачная модель для NER и классификации

Проект реализует совместное обучение (Multi-Task Learning) для одновременного решения двух задач:
извлечения именованных сущностей (NER) и многоклассовой классификации характеристик документа.

## Постановка задачи

В новостной аналитике часто требуется одновременно:
- **NER** — выделять именованные сущности (персоны, организации, даты и др.) на уровне токенов.
- **Классификация** — определять набор из 30 бинарных характеристик документа (например, `WORKPLACE`, `PARTICIPANT_IN`, `LOCATED_IN` и т. д.).

Классический пайплайн решает эти задачи раздельно, что ведет к дублированию признаков и неэффективному использованию вычислительных ресурсов. Проект проверяет гипотезу: **совместное обучение улучшает качество NER и стабильность классификации**.

## Данные

Используется публичный датасет [NEREL](https://huggingface.co/datasets/danasone/nerel) — российский корпус для NER и извлечения отношений.

- 58 уникальных NER-меток (BIO-разметка)
- 30 бинарных характеристик (multi-hot векторы)
- Разбивка: train 60% / val 20% / test 20%
- Фильтрация текстов длиннее 350 слов
- Аугментация обучающей выборки через token dropout (x3 копии)

## Архитектура модели

Класс `JointModel` — единый encoder с двумя головами:

```
Encoder (LaBSE_ner_nerel)
    |
    |--- [CLS]-токен --> CLS-голова (Linear, 30 выходов) --> BCEWithLogitsLoss
    |
    |--- Все токены  --> NER-голова (Linear, 58 выходов) --> CrossEntropyLoss
```

Потери двух задач объединяются через **автоматическое взвешивание на основе неопределенности** (uncertainty weighting): обучаемые параметры `log_sigma_token` и `log_sigma_cls` динамически балансируют вклад каждой задачи в суммарный лосс.

## Метрики

Целевые метрики на тестовой выборке:

| Метрика | Цель | Описание |
|---------|------|----------|
| `token_f1` | >= 0.50 | Macro F1 по NER-меткам |
| `cls_micro_f1` | >= 0.80 | Micro F1 по 30 характеристикам |

Обе цели достигнуты при пороге классификации `threshold = 0.50`.

## Инференс

```python
text = "Иван Иванов, сотрудник Газпрома, полетел в Париж 15 мая 2023 года."
ner_results, cls_results = predict(text)
```

Функция `predict()` возвращает:
- список пар `(слово, NER-метка)` для каждого токена
- список пар `(характеристика, вероятность)` для всех 30 характеристик

## Выводы

Гипотеза подтверждена: совместное обучение позволяет одной компактной моделью решать обе задачи, не жертвуя качеством ни на одной из них. Основные источники ошибок — длинные составные сущности, семантически неоднозначные токены и редкие типы сущностей, что типично для моделей, обученных на небольших датасетах.

Направления для улучшения:
- увеличение числа эпох обучения
- расширение обучающей выборки
- тонкая настройка гиперпараметров

## Рекомендации

**Данные**

> Датасет небольшой, и это главное узкое место. Самое очевидное — добавить реальные данные: другие русскоязычные NER-корпуса (например, Collection5, factRuEval), новостные тексты с ручной разметкой. Если новых данных нет, аугментацию можно усилить: в проекте используется только token dropout, но можно добавить замену синонимами через ruWordNet или back-translation.

**Дисбаланс классов**

> Редкие NER-теги (типа B-DISTRICT, O-ORDINAL) тянут метрики вниз. Предложение по борьбе: взвешенный CrossEntropyLoss с весами обратно пропорциональными частоте класса, focal loss, или oversampling примеров с редкими сущностями при формировании батчей.

**Архитектура**

> Можно попробовать mean pooling по всем токенам с учётом маски. Также стоит поэкспериментировать с добавлением cross-attention между двумя головами.
